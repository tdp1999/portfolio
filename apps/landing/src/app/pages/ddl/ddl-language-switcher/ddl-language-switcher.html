Six directions for the EN / VI toggle that lives in the header. Each demo is interactive — click to see active-state behavior. Pick by the weight you want to give bilingualism: visible-by-default (V1, V2, V5) treats VI as a first-class peer; menu-hidden (V3, V4) treats it as a setting; V6 is the most expressive, minimal one for a portfolio with little localized content yet.

Surveyed patterns from Stripe, Vercel, Linear, Apple, Notion, Mr.Doob, GitHub, and indie portfolios. Five distinct categories surfaced:

  • A · segmented Both languages always visible. Toggle-as-pill or text duo. Use when: bilingualism is a feature, both langs ~equally important. Variants V1, V2 below.
  • B · dropdown Active language as trigger, menu reveals all options. Standard pattern. Use when: 3+ langs, or when one lang is clearly primary. Variants V3, V4 below.
  • C · contextual A small "Read in [lang]" link surfaces only on pages where translation exists. Use when: translations are partial — VI only for /blog/* but not /uses. Akin to V5.
  • D · auto + override Browser Accept-Language picks initial, manual override in a settings page. Quiet, no header chrome. Skipped — no settings page exists yet; would be ghost code.
  • E · flag-based Country flags as chips. Rejected: politically fraught (VI flag ≠ all VI speakers; English is spoken in many countries). Industry consensus is to use language names, not flags.
v2 · mono inline toggle

Two lowercase mono codes separated by a slash. Active one in accent, inactive muted. Whole thing is one button — clicking anywhere flips. Weight: the lightest possible. Editorial vibe — fits the lowercase "tdp." mark.

v5 · single-click flip pill

One pill shows only the other language as a CTA-style chip ("Switch to VI"). Click flips immediately, no menu. Two states only — works perfectly for 2 languages. Weight: medium — explicit verb ("switch") reduces ambiguity vs. ad-hoc abbreviations. Trade-off: doesn't scale beyond 2 langs.

v6 · marquee swap (text crossfade)

Single mono code that swaps with a vertical text-roll animation on click — like Apple's "+ Pro / + Air" toggle or a flip-board departures sign. Quiet, opinionated, signals craft. Weight: light, but adds animation cost. Trade-off: the animation may feel showy on every page-load if remembered state changes.
