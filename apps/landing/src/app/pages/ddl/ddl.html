The DDL is where landing UI is designed, compared, and settled before it ships. Rather than scattering one-off mockups across branches and chat threads, every visual decision lives here as a page you can open, interact with, and link to. When a question comes up later — why does the hero stagger like that?, which filter-bar layout did we pick? — the answer is a URL, not an archaeology dig.

It is also the canonical reference for the things that must stay consistent: the type scale, spacing rhythm, colour and contrast, and the shared primitives. If a component appears here, this is how it is meant to look and behave.

The left sidebar groups every page by scope, from the smallest building blocks up to whole pages:

  • Foundations — tokens, typography, colour & contrast, layout container, prose rhythm.
  • Components — the reusable primitives (buttons, chips, inputs, nav, carousel, …).
  • Sections — composed blocks like the hero, selected-work, or get-in-touch.
  • Pages — full page templates (project detail, blog list & detail, about signatures).
  • Patterns — cross-cutting behaviours (interactions, pagination, fragment navigation, email).

Pick anything in the rail to open its page. The current page's headings appear in the right-hand “On this page” table of contents — use it to jump between sections.

Most pages follow the same anatomy, so once you know one you know them all:

  • A status chip by the title says where the design stands — exploring (still comparing directions), decided / shipped (a winner is in production), or deprecated (kept for history).
  • A decision record lists the directions that were weighed. When a winner exists it is marked with the reasoning; the alternatives that lost collapse into a “considered” disclosure so the page stays focused but the rejected options remain inspectable.
  • Page-scale demos carry a “⤢ Full width” button. The doc column is intentionally narrow for readability, which can squeeze a real page layout — that button pops the demo to the true viewport width so you judge it at production fidelity. Esc closes it.

Everything here obeys the landing system rules: the landing type scale only, spacing on the 4px grid, the four-breakpoint responsive system, and the shared landing-* primitives before any bespoke markup. When a UI decision changes, the DDL changes in the same commit — so what you see is always what ships.
