Vertical rhythm spec for long-form content. Every element below sits inside .landing-prose — what you see is the source of truth applied to project-detail, blog-detail, and any future long-form page.

Overview — h2 leading 72, trailing 16

First paragraph after the h2. Notice the gap between the heading and this paragraph is 16px, which feels tightly bound — the heading owns this paragraph. The h2's own 72px top margin creates the section break above (here against the first-child reset).

Second paragraph. Paragraph-to-paragraph rhythm is 16px (the previous paragraph's margin-bottom). Long-form reading at this rhythm scans naturally without feeling cramped.

Third paragraph to establish the default rhythm clearly. After this paragraph, an h3 will follow — note the leading-larger-than-trailing pattern means the h3 gets 48px top (collapsed with the prev p's 16 bottom → max wins → 48).

A sub-section — h3 leading 48, trailing 12

Paragraph immediately after the h3. Gap is 12px, even tighter than h2 since h3 represents a smaller break in flow.

An h4 — leading 32, trailing 8

Paragraph after h4. The hierarchy of breaks creates a clear visual outline: h2 = biggest section break, h3 = sub-section, h4 = paragraph-group label.

Content blocks — blockquote, figure, code

Each standout block declares its own halo. Margin collapsing handles the joining.

Blockquote with 24px top and bottom margin. Standout from running prose — italic Newsreader, accent border on the left. Use for direct quotes, callouts, or emphasized passages.

Paragraph after a blockquote. The 24 bottom of the quote + 0 top of this p → 24px gap. Larger than normal paragraph rhythm, signaling we're returning to flow after a pause.

Inline code looks like this — keyed Mono font, ink background, accent text. Block code follows:

Lists

Unordered list — 8px between items, 16px after the list as a whole:

  • First item, on the 24px indent.
  • Second item, with 8px above it from the previous item's margin-bottom.
  • Third item — long enough to wrap. Item text wraps consistently with the indent, the line-height of the body, and the rhythm of surrounding paragraphs.

Ordered list, same rhythm:

  1. Item one.
  2. Item two.
  3. Item three.

Above is an <hr> — 48px above and below, single horizontal rule in border color. Used for major thematic breaks within a long-form post.

Inline emphasis

Inline strong uses text-300 + weight 600 (slightly bolder against body color). Inline emphasis uses font-style italic — Inter italic by default. Inline link uses accent color with subtle underline (1px thick, offset 4px). Combine all three: strong with nested em and a link.

Back-to-back headings Edge case — h3 right after h2

When an h3 follows an h2 with no paragraph between them, the gap is max(h2.mb 16, h3.mt 48) = 48px. This is intentional — a "section header → sub-section header" sequence should feel like a tightly bound title block, not jarring.
