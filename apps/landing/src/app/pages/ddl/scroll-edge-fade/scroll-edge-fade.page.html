Scroll edge fade

Overflow affordance for any horizontally- or vertically-scrolling container. The landingScrollEdgeFade directive fades the edge(s) that have content out of view, so a clipped child reads as “scroll for more” rather than “cut off”. It also exposes its edge state + a scrollStep() method, so you can add explicit clickable chevrons. Resize the window narrow to see the strips overflow.

A · Fade only

Bare directive. Mask appears only when the strip overflows and dissolves at each end so the first/last item is never clipped.

@for (tab of tabs; track tab) { {{ tab }} }

B · Fade + chevron indicators

The shipped pattern (Selected Work tab strip): a gradient-scrim chevron fades the content INTO the arrow, so fade and affordance are one element. The strip sets [edgeFadeMask]="false" (the scrim is the fade); chevrons read rail.atStart() / rail.atEnd() to hide at the edges and call rail.scrollStep(±1) on click.

@for (tab of tabs; track tab) { {{ tab }} }

C · Vertical axis

edgeFadeAxis="y" on a fixed-height list — top/bottom fade as you scroll.

@for (row of rows; track row) {
{{ row }}
}

Usage

<!-- horizontal (default) -->
<div class="tabs" landingScrollEdgeFade>…</div>

<!-- with clickable chevrons -->
<button [hidden]="r.atStart()" (click)="r.scrollStep(-1)">‹</button>
<div landingScrollEdgeFade #r="scrollEdgeFade">…</div>
<button [hidden]="r.atEnd()" (click)="r.scrollStep(1)">›</button>

<!-- vertical -->
<div landingScrollEdgeFade edgeFadeAxis="y">…</div>
