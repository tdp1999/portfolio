Show more — clamp / scroll disclosure

A single reusable landing-show-more: it projects any content and, when that content exceeds maxHeight, either clamps it behind a See more / See less toggle (mode="toggle") or caps it and lets the overflow scroll inside the box (mode="scroll"). It is SSR / no-JS-safe (renders expanded on the server, collapses on hydration) and self-measuring — short content renders untouched with no chrome.

Toggle — long content clamps behind a button

mode="toggle" (default). Content over maxHeight clamps with a bottom fade; the toggle flips label + chevron. This is what the project-detail rails use.

The project-detail pattern below wide: two rails in a flex row with align-items: stretch. Scroll mode caps each at maxHeight, so a long column and a short column stay the same height instead of staggering.

Inputs

  • [maxHeight] — collapsed/cap height in px (default 300).
  • [mode] — 'toggle' (clamp + button, default) or 'scroll' (cap + internal scroll). Bind reactively to a breakpoint to swap behaviour by device.
  • [moreLabel] · [lessLabel] — toggle copy (default See more / See less).
  • [fade] — bottom gradient on the clamped content (toggle mode; default true).
