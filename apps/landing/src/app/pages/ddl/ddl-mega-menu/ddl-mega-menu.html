The "More" dropdown anchored to the nav. The current panel has 3 visible issues: title is center-aligned while description is left-aligned (CSS inheritance leak), the icon-frame box adds visual weight without earning it, and 5 items in a 2-col grid leaves the Resume cell stranded next to empty space. Below are 6 directions exploring different solutions — all rendered as always-open panels for side-by-side comparison.

Surveyed sites with notable mega-menus / nav dropdowns. Three style families emerged:

  • A · icon + text-column Side-icon row with title above description. Used by GitHub, Mintlify, Raycast. Iconography earns its weight when items overlap by name (Settings, Settings & billing, User settings…). For 5 distinct items like ours, the icon is decorative weight — but can be redeemed with tighter typography. (V1)
  • B · centered icon card Icon centered above title, all-center text in a card. Used by Vercel, Cal.com, Notion. Feels premium, doubles per-item height, and works best with bold colored icons + 2-3 cells in a row. With 5 items we'd need a hero row. (V2)
  • C · typography-only No icons. Bold title + 1-line description, or eyebrow + title. Used by Stripe, Linear, Apple docs. Editorial register; fits portfolios that favor type over chrome. Best when descriptions are confident one-liners. (V3, V4)
  • D · hero + list One item gets full-card hero treatment (the Resume CTA), others sit as compact rows. Used by Railway (the original reference — their "Enterprise" panel with the train illustration is the hero, the other items are compact cards). Solves the 5-items-in-2-col problem and signals what matters most. (V5)
  • E · inline-glyph reveal Icon-as-inline-glyph (text-sized, monochrome). Description hidden by default, surfaces on hover. Used by Vue docs, Astro, indie portfolios. Dense, quiet — earns the icon by treating it as a tiny typographic accent rather than a card element. (V6)

NN/g findings worth respecting: (1) mega-menus work because users see options at a glance — don't hide the description for items where it's load-bearing. (2) hover-open is consistently frustrating — click-to-open + click-outside + Esc is the gold-standard. (3) don't take over the whole viewport — keep panel ≤ 560–640px wide for 1-col, ≤ 720px for 2-col with hero.
Approaches explicitly rejected: flag/illustration grid (overkill for 5 utility links), full-bleed slide-down panel (too aggressive for a portfolio), country/locale picker (the language switcher handles this separately).
