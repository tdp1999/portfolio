Notes

  • All three variants share a left-column rebuild: 3 split-intent CTAs replace the single "Send a message" button. Long-form phrasing (one full clause per CTA) gives the user a verbal off-ramp instead of a generic "send".
  • Channel row (GitHub · LinkedIn · X · Resume) is removed from all variants — duplicates the footer and §3 bio card. Resume relocates via the CV placement strip below the variants.
  • V1 (SVG globe) — static constellation. ~8kb. Cheapest cost, strongest narrative compression (HCMC + SGN + SYD as one image). Pulsing nodes draw the eye without rotating geography.
  • V2 (drag-rotate) — adds interactivity. The drag UX is the differentiator vs V1; in production this is the slot for globe.gl WebGL with atmosphere + texture. Risk: feels like an unnecessary toy for a contact section, and mobile drag is awkward.
  • V3 (info panel) — no illustration. Strongest signal density (status + timezone + response window + served-from). Best for "hiring manager scanning" use case. Risk: less memorable visually.
  • CV placement — (a) gives the document its own ceremony but adds a new row; (b) keeps the single-CTA-stack rhythm but flattens the resume's weight; (c) anchors the right column but creates competition with the globe/info above.
  • Recommendation pending visual review. V1 + CV (a) is the safest aesthetic floor. V3 + CV (b) is the most information-efficient. V2 only if drag interactivity earns its keep on first impression.
