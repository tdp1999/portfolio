When to pick each magazine variant

V1 — Center hero + 4 side

Magazine-cover energy. Use when one post genuinely deserves to be the front page and four supporting stories carry equal weight. The hero embeds its cover as an inline figure (no top cover) so the title earns the first read.

V2 — Left hero + 4 right

Editorial pacing — L → R reading order. The split hero (cover left, content right) compresses the hero vertically; the 2×2 right grid catches the eye on the rebound. Best when the side cards are siblings of equal weight.

V3 — Top hero + 6 archive

Density-forward. A wide split hero on top + 3×2 archive grid below. Pick this when the featured pool is large and rotates often — six supporting slots accommodate breadth without forcing vertical sprawl. Archive cards are compact (no excerpt by default).

Page-level constraints

Featured strip occupies ~800-900px

Baseline: 1920×1080 desktop, ~180px of browser chrome → ~900px usable. The page hero (reused from /projects) is compact on /blog so the featured strip can earn ~800-900px and read as a real magazine spread on first scroll. The list section opens below the fold by design.

Title font is sans-serif medium

Magazine variants use the body font (Inter) at weight 500 for titles instead of the display serif. The serif is reserved for prose body inside the post detail page. List rows + grid cards on this page also use the medium-weight sans for consistency.

List reuses /projects

Below the strip, the list section reuses the /projects shape verbatim: landing-results-count · landing-view-toggle · landing-filter-chip pills · landing-segmented sort · landing-pagination. The one new addition is a debounced search input (landing-input, 250ms).
