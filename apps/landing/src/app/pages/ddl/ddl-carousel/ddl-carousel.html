A single reusable landing-carousel: touch swipe (native snap), mouse drag, ←/→ keys, prev/next arrows, dots, optional thumbnails / peek / loop, and per-slide landing-figure caption + FIG number. It is not responsive by itself — when a layout needs a slider on small screens, the consumer swaps components by breakpoint (Selected Work keeps the curated grid on laptop+ and swaps to this below).

Swipe / drag / arrow-keys / click arrows or dots. Arrows fade out at the ends.

A visible edge of the next frame makes "there's more" obvious without controls. Pop the demo to full width to see the true peek sliver — the doc column crops it.

A filmstrip replaces the dots when the shots are distinct and jumping around matters.

Next from the last slide wraps to the first; arrows never disable.

Pass [items] + an <ng-template landingCarouselSlide> to project arbitrary content per slide. The carousel still owns track / snap / peek / dots / arrows; the consumer owns what's inside. Used by the blog featured strip below tablet (lead hero stays, secondary cards swipe).
