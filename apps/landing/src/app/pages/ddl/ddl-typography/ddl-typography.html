One serif (Newsreader) for editorial moments, one sans (Inter) for everything readable, one mono (JetBrains) for labels and metadata. A single text node should always have one obvious owner — never reach for "looks nice", reach for the role.

Three voices, three jobs

newsreader · display

Aa

Use for: hero h1, page titles, pull-quotes.

Don't: buttons, body, repeated card titles.

inter · body / ui

Aa

Use for: paragraphs, buttons, links, card bodies — anything the reader scans.

Don't: tags, captions, hero phrase.

jetbrains mono · meta

Aa

Use for: chips, eyebrows, captions, code, dates, status labels.

Don't: full sentences, paragraphs.

element default · outside prose

  • Bare <h1> → display-lg (Page h1). Hero overrides via utility class to xl.
  • Bare <h2> → display-md (Sub-hero).
  • Bare <h3> → display-sm.
  • Bare <h4> → body-xl (fluid 16→20). Below display scale.

prose override · inside .landing-prose

  • Font switches from Newsreader → Inter 600. Newsreader stays for h1 page titles OUTSIDE prose (e.g. project header).
  • Every level downsizes one step: h1 → md, h2 → sm (Section h2 role), h3 → body-xl, h4 → body-lg.
  • Long-form reading doesn't compete with display-scale headings.
  • See /ddl/prose-flow for full spec.

FIG. 01 · canonical pairing

pairing rules

  1. One display moment per view. A page gets one Newsreader headline. Outside prose (page titles, hero, project header) Newsreader carries display headings. Inside .landing-prose editorial columns, sub-headings (h2/h3/h4) switch to Inter 600 for sustained readability.
  2. Eyebrow above display. Mono caps eyebrow + Newsreader headline is the canonical pattern (hero, section openers, project detail header).
  3. Chips never under headings of the same type. A chip cluster (mono caps) belongs in card meta or under a project title — not directly under another mono label.
  4. Pull-quote interrupts body. When using landing-pull-quote inside reading copy, leave 24px padding above/below; the indigo border carries the "this is a moment" cue.
  5. Captions are always mono. Any text labeling a media element (image, code block, diagram) is Mono caps — not Inter italic, not Newsreader.

quick decision table

Question Use
Is this a page-defining headline or hero phrase? Newsreader
Is this a paragraph the reader is meant to read fully? Inter
Is this a button, input, link, or card title? Inter
Is this labeling, tagging, or pointing at something? Mono
Is this a short ALL-CAPS metadata string? Mono
Is this code or a file path? Mono
Is this a quote inside reading copy? Newsreader italic
